# Обязательный алгоритм изменений (для агента)

Этот алгоритм обязателен для каждого изменения в проекте после получения одобрения от пользователя.

1. Внести кодовые изменения по задаче.
2. Попросить одобрение пользователя на фиксацию результата (если требуется по контексту).
3. После одобрения обязательно обновить документацию:
   - ТЗ/описание актуального поведения в `README.md`;
   - план и статус выполнения в `PROJECT_PLAN.md`.
4. Проверить рабочее дерево (`git status`) и убедиться, что код и документы синхронизированы.
5. Создать коммит с осмысленным сообщением.
6. Отправить изменения в удаленный репозиторий (`git push origin main`).
7. Сообщить пользователю:
   - какие файлы ТЗ/плана обновлены;
   - хеш коммита;
   - статус push.

Нарушение алгоритма недопустимо: изменения считаются незавершенными, пока не обновлены `README.md`, `PROJECT_PLAN.md` и git.
