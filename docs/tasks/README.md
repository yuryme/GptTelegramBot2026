# Хранение ТЗ и согласований

В этой папке хранятся все рабочие ТЗ, согласования и постановки задач между участниками проекта.

Цель:
- не пересылать копии ТЗ по чатам;
- хранить одну актуальную версию документа в проекте;
- быстро понимать, кто автор, кому адресован документ, когда он создан и какой у него статус.

## Структура

- `INDEX.md` — единый реестр документов.
- `active/` — актуальные документы, по которым идет работа.
- `archive/` — завершенные или устаревшие документы.
- `templates/` — шаблоны для новых ТЗ и согласований.

## Правило именования файлов

Формат имени файла:

`YYYY-MM-DD_from_to_topic.md`

Примеры:

- `2026-03-09_customer_to_manager_functional_status.md`
- `2026-03-09_manager_to_architect_p1_scope.md`
- `2026-03-09_customer_to_manager_iteration_01_temporal_normalization.md`

## Что обязательно должно быть в документе

В начале каждого документа должны быть:

- дата;
- автор;
- адресат;
- тема;
- статус (`draft`, `approved`, `in_progress`, `done`, `archived`);
- краткая цель документа.

## Как пользоваться

1. Создаем новый документ в `active/` по шаблону.
2. Добавляем запись в `INDEX.md`.
3. В чате передаем не содержимое ТЗ, а путь к файлу.
4. После завершения работы переносим документ в `archive/` и обновляем `INDEX.md`.

## Правило проекта

Если появляется новое ТЗ, согласование или постановка задачи для конкретной роли, оно должно быть зафиксировано здесь, а не только в переписке.
